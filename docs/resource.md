# 推荐你看的一些资料

## 首先推荐：《提问的智慧》

从高中到大学，一个很重要的变化是：很多事情不会再有人一步一步告诉你该怎么做。你需要逐渐学会主动查资料、判断信息、提出问题，并把自己的问题表达清楚。

我推荐大家阅读《提问的智慧》这篇文章。虽然它最初主要面向技术社区中的提问场景，但其中很多原则同样适用于大学学习和日常沟通。

它提醒我们：一个好的问题，往往不是随口问出来的，而是在清楚描述背景、尝试自行解决、整理已有信息之后形成的。学会提问，本质上也是在训练自己的思考能力、信息检索能力和表达能力。

进入大学后，你会遇到很多需要向老师、助教、学长学姐、同学或网络社区求助的场景。提前理解“如何把问题问清楚”，会让你在学习、科研、社交和解决实际问题时更加高效，也更容易获得他人的有效帮助。

## 第二个推荐：上海交通大学生存手册

我很推荐大家阅读《上海交通大学生存手册》。虽然它的名字里写着“上海交通大学”，但其中很多内容并不只适用于交大的同学，而是对大多数即将进入大学的同学和在大学就读的同学都有参考价值。

这份手册可贵的地方在于，它没有只介绍“大学生活有多精彩”，而是比较直接地讨论了很多新生真正会遇到的问题：如何适应大学的学习方式，如何看待失败和压力，如何避免被不合理的评价体系裹挟，如何理解绩点、选课、转专业、保研、科研、工作和未来发展。

进入大学后，很多问题并不会立刻显现。你可能一开始只是觉得课程难、信息多、选择杂，但过一段时间之后，会逐渐发现：真正影响大学体验的，往往不是某一次考试或某一门课，而是你如何理解自己、如何安排时间、如何判断信息、如何面对比较、如何在复杂环境中保持清醒。

《上海交通大学生存手册》提供的不是标准答案，而是一种很有价值的提醒：大学不是简单地沿着高中惯性继续往前走，而是需要你逐渐学会主动选择、独立判断，并为自己的选择承担后果。

当然，这份手册中的部分内容会带有交大的具体语境，阅读时不必完全照搬。但我仍然建议大家认真看一遍，尤其是关于学习方式、信息素养、时间管理、绩点、科研和未来规划的部分。哪怕你暂时还用不上，提前留下印象，也可能在以后遇到类似问题时帮你少走一些弯路。

## 第三个推荐：CSDIY.WIKI

如果你是计算机相关专业的同学，或者对计算机学习感兴趣，我很推荐你阅读 [CSDIY.WIKI](https://csdiy.wiki/)。

这是一份非常系统的计算机自学指南，里面整理了很多计算机学习资源、学习路线、经典课程、常用工具和自学经验。对刚进入大学、还不太清楚计算机专业该怎么学的同学来说，它可以帮助你更早建立一张相对完整的学习地图。

我自己在暑假阅读这份指南时收获很大。它让我意识到，大学里的学习不应该只停留在完成课程作业和应付考试上，还可以主动利用公开课程、开源社区、技术文档和项目实践去拓展自己的能力。后来我整理这个准大学生入学指南，也在一定程度上受到了 CSDIY.WIKI 的启发。

不过，也需要提醒大家：CSDIY.WIKI 的内容比较丰富，里面提到的课程和资料不需要一口气全部学完。刚开始阅读时，可能会觉得信息量很大，甚至产生“自己是不是落后很多”的感觉。其实完全没有必要这样想。它更适合作为一个长期参考的资源库，而不是一份必须立刻完成的任务清单。

如果你是计算机专业新生，可以重点关注它里面关于计算机学习路线、编程工具、经典课程和自学方法的部分。如果你不是计算机专业，也可以把它当作了解信息素养、工具使用和自学方式的参考，不必完全照着计算机专业路线学习。

大学阶段很重要的一点，是逐渐学会主动寻找高质量资料，并判断哪些内容适合自己当前阶段。CSDIY.WIKI 的价值不只是提供课程链接，更重要的是展示了一种主动学习、持续积累、善用公开资源的学习方式。

[在线阅读：CSDIY.WIKI](https://csdiy.wiki/)


## 第四个推荐：各学校的培养方案和课程计划

除了看专业介绍，我也非常建议大家去查目标学校和目标学院的培养方案、课程计划或本科生培养手册。

专业名称只能告诉你一个大概方向，真正决定你大学四年主要学什么的，是培养方案。它通常会写清楚这个专业的培养目标、毕业要求、课程结构、学分要求、必修课、选修课、实践环节、毕业设计以及可能的专业方向。

如果你有转专业或保研的打算，更应该提前关注培养方案。对于转专业的同学来说，大一上的培养计划决定了你想要达到高绩点需要卷那几门课，暑假请把重心放在这几门课上面，并且在转专业完成之后往往需要补修部分课程；保研也通常要求前三年课程按计划完成。提前了解这些要求，可以帮助你少走一些弯路。

培养方案不一定好读，但它比很多经验帖更接近专业本身。看不懂每门课没关系，先大致看清这个专业到底学什么、重视什么能力、和自己想象中是否一致，就已经很有帮助。
